# EduAssist — внутренняя документация команды

> Хакатон | 1-2 дня | Команда: 1-2 чел (full-stack)  
> Стек: React · FastAPI · PostgreSQL · MinIO · Yandex AI Studio (YandexGPT + SpeechKit)

---

## Суть продукта

**EduAssist** — AI-платформа для школы с тремя ролями: директор, учитель, ученик.  
Алиса AI — не чат, а рабочий инструмент: проверяет работы, анализирует урок, строит расписание.

---

## Роли и доступ

| Роль | Что может |
|------|-----------|
| **Директор** | Генерация расписания из Excel, просмотр метрик (шум, оценки) по всем классам |
| **Учитель** | Проверка домашек, запуск анализа урока, просмотр профилей учеников |
| **Ученик** | Загрузка домашних работ, объяснение тем через Алису (наводящий режим) |

---

## 4 ключевых фичи (все в MVP)

### 1. Проверка домашних работ
- Ученик загружает PDF / фото тетради → MinIO
- Backend: OCR через Yandex Vision → текст
- YandexGPT анализирует: ошибки, оценка, 3 уточняющих вопроса
- Учитель видит структурированный отчёт

**Function call:** `checkHomework(student_id, file_url)`

---

### 2. Анализ шума / урока в классе
- Учитель нажимает «Начать анализ» → браузер пишет аудио (Web Audio API)
- Каждые N секунд → Yandex SpeechKit STT → текст
- При высоком уровне шума → более частая запись + мягкий сигнал (UI-уведомление)
- Голос учителя фильтруется по паттернам (ключевые слова команд, тональность)
- По итогу урока: тезисная сводка + график активности → директор + учитель

**Function call:** `analyzeLesson(class_id, audio_chunk)`

> ⚠️ Голос учителя: Алиса реагирует только на явные триггер-фразы («Алиса, ...»), иначе — фоновый режим записи шума.

---

### 3. Генерация расписания (роль директора)
- Директор загружает Excel: учителя, классы, предметы, ограничения
- YandexGPT (function calling) парсит данные, выявляет конфликты
- Генерирует расписание с учётом правил:
  - нет «окон» у учителя (пара в 10:00, потом перерыв до 18:00 — запрещено)
  - не более N пар подряд
  - предпочтения преподавателей
- Результат: таблица + возможность скачать Excel

**Function call:** `generateSchedule(teachers[], classes[], constraints{})`

---

### 4. Персональный профиль ученика
- В БД (PostgreSQL) хранятся: все сданные работы, оценки, выявленные слабые темы
- Алиса при объяснении темы тянет профиль → адаптирует объяснение
- Учитель видит прогресс + топ слабых тем по классу
- **Наводящий режим**: Алиса не даёт готовый ответ — задаёт встречный вопрос

## Архитектура (упрощённо)

```
React (SPA)
    │
    ├── Auth (JWT, роль в токене)
    │
FastAPI
    ├── /api/homework     → Yandex Vision OCR + YandexGPT
    ├── /api/lesson       → SpeechKit STT + YandexGPT
    ├── /api/schedule     → YandexGPT function calling
    └── /api/profile      → PostgreSQL CRUD
    │
MinIO (файлы: PDF, фото, аудио)
PostgreSQL (users, works, grades, schedule, lesson_logs)
```

## Yandex AI Studio — что используем

| Инструмент | Где |
|-----------|-----|
| YandexGPT | Проверка ДЗ, генерация вопросов, расписание, объяснения |
| Yandex SpeechKit STT | Анализ аудио урока |
| Yandex Vision OCR | Фото тетрадей → текст |
| Function Calling | checkHomework, generateSchedule, analyzeLesson |
| Хранение контекста | Профиль ученика передаётся в промпт как system context |

## Риски и упрощения для MVP

| Риск | Решение |
|------|---------|
| Фильтрация голоса учителя сложная | MVP: фиксируем уровень шума (децибелы), STT только для сводки |
| Расписание — NP-задача | Ограничиваем входные данные, GPT генерирует первый вариант, директор правит |
| OCR рукописного текста | Yandex Vision справляется с печатью хорошо; рукопись — stretch goal |
| Нет времени на дизайн | Используем shadcn/ui или Ant Design — готовые компоненты |
